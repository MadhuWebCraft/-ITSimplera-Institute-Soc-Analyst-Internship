
# 🛡️ Week 2 – pfSense Firewall Deployment and Configuration

## ITSimplera Institute – SOC Internship

## 📌 Overview

This repository contains my **Week 2 Internship Task** completed as part of the **ITSimplera Institute SOC Internship Program**. The objective was to deploy and configure **pfSense Firewall** to create a secure network environment by implementing firewall security policies, website filtering, GeoIP-based blocking, and monitoring firewall activity.

---

# 🎯 Objectives

* Deploy the pfSense Firewall virtual appliance.
* Configure the pfSense Firewall.
* Install and configure the **pfBlockerNG** package.
* Implement **GeoIP-based country blocking**.
* Configure **DNSBL** to block selected websites.
* Create administrator access rules for the pfSense WebGUI.
* Monitor firewall logs and pfBlockerNG reports.
* Validate firewall events and security policies.

---

# 🛠️ Technologies Used

* pfSense Firewall
* pfBlockerNG
* VMware Workstation
* Windows 10 Virtual Machine
* Kali Linux Virtual Machine
* DNSBL
* GeoIP Filtering

---

# 📂 Project Workflow

### Step 1 – Deploy pfSense Firewall

* Created a pfSense virtual machine.
* Configured WAN and LAN interfaces.
* Completed the initial installation.

### Step 2 – Configure Firewall

* Accessed the pfSense WebGUI.
* Configured hostname, DNS, LAN, WAN, and administrator credentials.

### Step 3 – Install pfBlockerNG

* Installed the pfBlockerNG package using the Package Manager.
* Enabled DNSBL and IP filtering features.

### Step 4 – Configure GeoIP Blocking

* Created GeoIP rules.
* Configured country-based traffic filtering.

### Step 5 – Configure DNSBL

* Enabled DNSBL.
* Added custom blocked domains.
* Applied DNSBL updates.

### Step 6 – Website Blocking Validation

* Tested blocked websites.
* Verified DNSBL successfully prevented access.

### Step 7 – Configure Firewall Rules

* Created administrator access rules.
* Secured the pfSense WebGUI.

### Step 8 – Firewall Monitoring

* Reviewed firewall logs.
* Verified allowed and blocked traffic.

### Step 9 – pfBlockerNG Reports

* Monitored DNSBL alerts.
* Verified blocked domains and filtering statistics.

### Step 10 – Final Verification

* Confirmed firewall services were running successfully.
* Verified security policies were functioning correctly.
